Caugia V2 Component Library

This page shows the core components used across Caugia V2. Each block is both a visual example and a structural reference for future work.

Layout primitives

Layout is based on a small set of predictable building blocks.

  • header with fixed position, blur and border
  • .container for horizontal centering and max width
  • .wrap to add vertical spacing under the fixed header
  • section elements for vertical rhythm
  • .card for framed content blocks

Hero pattern

Used on the homepage and most top level pages. One primary heading and one calm supporting line.

Example hero title

A calm, concise line that sets context. No promises that cannot be kept. No unnecessary decoration.

Cards

Cards are the primary way to present structured information. They are used for explanations, groupings, modules and CTAs.

Standard card

This is a standard card. It uses the base card component with calm spacing and clear hierarchy.

  • Used for sections on About, Services, Resources
  • Suitable for concise blocks of copy
  • Always kept simple and structured

CTAs

Calls to action are short, clear and direct. They invite the next step without pressure.

Primary CTAs

Use for key actions such as the GTM Score, the GTM Intelligence Report and contact.

Grids

Grids are used for service lines, modules and similar sets of blocks. The same responsive pattern is reused across the site.

Grid item one

Short explanatory text. One or two sentences.

Grid item two

Same height behaviour on desktop where possible.

Grid item three

On small screens the grid falls back to one column.

Module detail layout

Modules reuse the same vertical structure. Each module page is made of five main sections.

  1. Hero with pillar and GRIP context
  2. What is inside this module
  3. Capabilities assessed
  4. Common failure patterns
  5. Impact on the GTM system and CTAs

Content stays short, direct and system oriented. The goal is clarity, not persuasion.

Typography

Headings and body copy are simple and consistent.

H1 title example

Used once per page for the main title. Clear, specific and short.

H2 section heading example

Used for major sections inside a page and inside cards.

Standard paragraphs use a readable line height and moderate width. Lists are used when structure helps understanding. Text avoids excessive qualifiers and marketing language.

Lists

Lists are used to keep information scannable. They are especially important in diagnostic content.

Unordered list

  • Clear and simple items
  • Used for capabilities, failures and impacts
  • No nested lists unless necessary

Ordered list

  1. Used for sequences or processes
  2. Numbered when order matters
  3. Short and direct wording

Form pattern

Forms are minimal and use clear labels. No complex layout or decorative elements.

Page level CTA section

Many pages close with a calm CTA block. It usually contains one or two short paragraphs and up to three CTAs.

Example CTA block

Start with a short summary of the next logical step. Then offer concrete options.

Notes for future development

  • Reuse header, navigation and footer as they are in this version
  • Do not introduce new visual patterns without mapping them to this library
  • Keep copy calm, concrete and free from hype
  • Use cards and grids instead of new layout constructs
  • Respect the existing spacing and typography system
